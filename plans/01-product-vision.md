# Product Vision

## Working Name

ActionOS

## Vision

ActionOS is a shared operating layer for team work.

It combines flexible boards, task management, meetings, decisions, dashboards, and automation into one workspace. It should feel powerful like Monday.com, but more focused on execution after meetings.

## Positioning

ActionOS is not only a task list.

It is not only a meeting notes tool.

It is not only a project board.

It is a system for turning team conversations, plans, responsibilities, and decisions into trackable work.

## Product Promise

ActionOS helps teams move from:

- Scattered notes to assigned action items.
- Verbal promises to visible commitments.
- Static boards to living workflows.
- Meetings with no follow-up to accountable execution.
- Personal task lists to shared team operating rhythm.

## Core Pillars

### 1. Flexible Boards

Boards are the foundation. A board can represent tasks, projects, meetings, bugs, clients, goals, requests, approvals, or any custom workflow.

### 2. Multi-User Workspaces

Everything belongs to a workspace. Workspaces contain members, teams, boards, dashboards, meetings, permissions, templates, and activity.

### 3. Meeting-to-Work Engine

Meetings should produce structured output:

- Notes
- Decisions
- Tasks
- Follow-ups
- Blockers
- Reminders

The meeting is not the end of the workflow. It is one source of work.

### 4. Personal Command Center

Each user gets a "My Work" page that shows:

- Assigned tasks
- Overdue work
- Upcoming work
- Blocked items
- Meeting follow-ups
- Watched items
- Recently mentioned items

### 5. Team Accountability

Managers and team members should quickly see what is moving, what is stuck, and what was promised.

### 6. Automation Layer

Later versions should support rules such as:

- When status changes, notify someone.
- When a due date approaches, create a reminder.
- When a meeting ends, create follow-up tasks.
- When an item is blocked, escalate it.

## "Monday On Steroids" Meaning

For ActionOS, "on steroids" means:

- Flexible boards and custom fields.
- Rich meeting follow-up workflows.
- Decision tracking.
- Commitment tracking.
- Cross-board relationships.
- Team dashboards.
- Automation.
- Strong integration into the existing Fritz ecosystem.

## Product Personality

ActionOS should feel:

- Clear
- Fast
- Accountable
- Serious enough for business use
- Flexible enough for many workflows
- Focused on execution, not decoration

The UI should be work-focused, dense enough for repeated daily use, and easy to scan.
